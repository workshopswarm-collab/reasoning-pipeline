# Research Vault

This vault is the canonical human-readable research memory for the reasoning pipeline.

Design goals:
- human-readable hierarchical knowledge base
- retrieval-friendly structure for QMD
- provenance-aware research artifacts
- support retrospective pipeline improvement
- avoid forecast-lifecycle coupling

Top-level sections:
- `00-system/` — methodology, templates, taxonomies, evaluation criteria
- `10-domains/` — subject-area knowledge
- `20-entities/` — reusable dossiers on recurring actors/objects
- `30-drivers/` — reusable causal mechanisms
- `40-research/` — source notes, agent findings, investigations, syntheses, assumptions
- `50-retrospectives/` — evaluation of input quality, agent performance, and method changes
- `70-inbox/` — temporary holding area for uncategorized or not-yet-stable material

Design rule:
- keep folders coarse and stable
- push nuance into frontmatter metadata and QMD retrieval
- treat canonical dossiers as slow-moving memory; route new information into research artifacts first
