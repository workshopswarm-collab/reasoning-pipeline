# Source reliability

This folder is subordinate to `qualitative-db/00-system/README.md`.

Use this folder for **short reference notes about source classes or metric-source families**.

These are support documents, not primary operating rules.

## Fresh-instance shortcut

Read this folder only when:
- you need guidance on a metric-source class
- you are unsure how to treat a sports/crypto metric source family
- a task specifically depends on source-quality distinctions

Do **not** read this folder by default before every task.

## What belongs here

Good fit:
- source-family notes
- metric-source guidance
- quick reminders about common source strengths or weaknesses

Not the main purpose of this folder:
- case-specific source extracts
- evidence from a live investigation
- full methodology rules for canon vs research

Those belong elsewhere:
- case-specific source notes -> `qualitative-db/40-research/cases/<case-key>/researcher-source-notes/`
- operating rules -> `qualitative-db/00-system/README.md` and related governance/methodology files

## Current usage

The files here are lightweight reference notes.

Treat them as optional support material, not mandatory onboarding.
