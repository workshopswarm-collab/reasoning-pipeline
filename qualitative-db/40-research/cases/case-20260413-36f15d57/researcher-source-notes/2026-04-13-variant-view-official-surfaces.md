# Summary

Official DeepSeek public surfaces checked on 2026-04-13 still center the publicly accessible flagship around V3/V3.2 rather than any V4-labeled launch. This matters because the contract requires a public, clearly defined next-major V release, not mere rumor or internal readiness.

## Key facts extracted

- The official DeepSeek homepage prominently advertises DeepSeek-V3.2 and links users to chat, app, API platform, and API docs.
- The homepage research links include DeepSeek R1, DeepSeek V3, DeepSeek V2, and related model families, but no visible DeepSeek V4 link on the checked surface.
- The API docs front page states that `deepseek-chat` and `deepseek-reasoner` correspond to DeepSeek-V3.2.
- The API docs news index lists releases through DeepSeek-V3.2, including V3-0324 and R1-0528, but no V4 item on the checked surface.

## Evidence directly stated by source

- Homepage language: "DeepSeek-V3.2 正式版发布" with broad availability language for web, app, and API.
- API docs: "The `deepseek-chat` and `deepseek-reasoner` correspond to the model version DeepSeek-V3.2".
- API docs news list includes V3.2, V3.2-Exp, V3.1, R1-0528, V3-0324, and earlier releases.

## What is uncertain

- These checked surfaces do not prove a V4 launch is impossible before deadline; they only show that as of the verification time the official public product/docs surfaces still present V3.2 as the current public model family.
- A separate official announcement channel (for example X, GitHub, WeChat, or blog/news post) could still announce a qualifying V4 later before deadline.

## Why this source may matter

The contract's primary source of truth is official DeepSeek information. Official public product and documentation surfaces are therefore highly relevant to whether a qualifying public release has actually happened.

## Possible impact on the question
